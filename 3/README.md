У першому рефакторінгу, було створено абстрактний базовий клас Shape, який визначає метод calculate_area як абстрактний. Потім були створені конкретні класи Circle, Rectangle і Triangle, які успадковують від Shape і реалізують власну логіку розрахунку площі. Це дозволяє використовувати поліморфізм і замінити оператор switch на створення об'єкта відповідного класу фігури.

У другому рефакторингу, логіка розрахунку площі для кожного типу фігури зберігається в окремих методах класу ShapeCalculator. Метод calculate_area тепер приймає тип фігури як аргумент і викликає відповідний метод для розрахунку площі. Це дозволяє уникнути використання оператора switch і забезпечує більш чітку та модульну структуру коду.
